### 2026-06-30 тАФ Broad USB detection + state classification (CL-031)

Old `_detect_usb_devices` only found USBs whose label was literally "Ventoy" тАФ
a chicken-and-egg problem for first-time setup (a blank USB couldn't be
detected as a target for the very Ventoy install). Replaced with a three-method
all-USB scanner plus a classifier so the mode prompt and setup wizard both see
**any** plugged-in drive.

- **`_detect_all_usb_devices()`** тАФ new primary detector. Three signals:
  1. `lsblk -d -n -b -o NAME,TRAN,SIZE` for `TRAN=usb` (authoritative on
     Linux). Size > 0 required so empty card readers / hub placeholders are
     skipped.
  2. `/sys/block/<dev>/removable == 1` for USB-attached drives behind bridges
     that hide the TRAN hint. Optical (`sr0`) and 0-sector entries skipped.
  3. `udevadm info --query=property` for `ID_BUS=usb` as a final safety net
     for hot-plugged drives. Also size-guarded.
- **Root disk exclusion** тАФ `findmnt -no SOURCE /` + `lsblk -no PKNAME`
  identifies the disk backing `/` and removes it from candidates so a
  mis-flagged SATA bridge never lands the host root in the USB list.
- **`_classify_usb_device <dev>`** тАФ returns one of:
  - `ventoy` (has Ventoy or VTOYEFI label)
  - `formatted` (ext/exfat/vfat/ntfs/btrfs/xfs/f2fs present, but not Ventoy)
  - `blank` (no recognized filesystem тАФ ready for fresh Ventoy install)
- **`_detect_usb_devices_ventoy()`** тАФ kept for places that explicitly need
  Ventoy-only (auto-detect-on-startup uses this; setup wizard / mode prompt
  use the broad detector).
- **`_select_uca_mode` prompt now lists each USB with its classification** so
  the operator sees at a glance "BLANK тАФ option 1 will install Ventoy" vs
  "VENTOY тАФ ready to use" vs "FORMATTED тАФ option 1 will REFORMAT". The
  USB-mode summary line adapts text accordingly.
- **`_setup_device_interactive` (menu option 11)** lists every USB with size,
  label, and classification badge; manual entry and "no USB" fallback retained.
  Dead "no Ventoy detected" branch removed (now unreachable since the broader
  scan + classification covers every state).
- **Live-tested with two USBs plugged in simultaneously:** `/dev/sdb` (empty
  card reader, 0B) correctly filtered; `/dev/sdc` (29.5G Ventoy) detected and
  classified `ventoy`. Root disk `/dev/sda` correctly excluded.

### 2026-06-30 тАФ UCA_MODE selector + USB-first install routing (CL-030)

The platform now refuses to silently mutate the host when an operator is
managing the USB. Top-level switch:

- **Boot-time mode selector** тАФ `_select_uca_mode()` runs immediately after
  environment resolution in `main()`. Silent USB scan first; **no USB тЖТ auto
  `UCA_MODE=host`** with no prompt. **USB present тЖТ prompt** the operator to
  choose:
  - `1) USB` тАФ install persistence/SSH/firewall/boot bridge/aliases/bash
    profile to the USB; host stays untouched after initial bridge writes;
    compute-only host bridge.
  - `2) HOST` тАФ local enhancements only; USB never touched.
- **CLI bypass**: `--mode usb|host`, `--usb`, `--host` shorthand, or env
  `UCA_MODE=usb|host` (also documented at the top of `usb/env.example`).
- **`_uca_install_root()` helper** added to `usb/lib/core.sh`. Returns the
  resolved root: `<usb-mount>/usb-hemlock/etc/uca/` (USB mode) or
  `$XDG_CONFIG_HOME/usb-compute-automation/` (host mode). Auto-creates the dir.
- **`UCA_PERSISTENCE_PATH`** is pre-resolved and exported by `_select_uca_mode`
  so spawned child scripts can compute the install root without re-running
  detection (`_uca_primary_persistence` is a menu.sh-local function).
- **Menu pruning** тАФ in `UCA_MODE=host`, options 1, 2, 6, 8, 9, 11, 12, 17, 19
  (USB setup, usbctl, automount, startup wizard, persistence, device
  setup/config, USB access, Hemlock) are HIDDEN from both the text and
  whiptail menus. Whiptail title shows `USB-Hemlock тАФ Mode: {USB|HOST}`.
- **Install routing wired into the four scripts that previously hardcoded
  `$HOME`:**
  - `alias_manager.sh`: `ALIAS_FILE` resolves via `_resolve_alias_paths()`
    тЖТ `<root>/bash_aliases.sh` + `<root>/alias_backups/`.
  - Bash Profile Manager (menu opt 10): copies `bash_enhanced.sh` into
    `<root>/bash_profile.sh`; host `~/.bashrc` gets a *single* one-line
    `source` (the explicit "initial bridge write" allowance).
  - `install-antivirus.sh`: `LOG_DIR` + `QUARANTINE_DIR` redirect to
    `<root>/var/{log,quarantine}/system_toolkit/` in USB mode; runtime
    `virus` binary gets the routed paths substituted via `sed -i` after
    heredoc emission so cron jobs talk to the right location. Permissions
    on `QUARANTINE_DIR` changed `0700` тЖТ `0750` (CL-017 compliance тАФ was a
    pre-existing violation).
  - `clean-local.sh`: **hard-blocks** with exit 12 + `[BLOCKED]` message
    when `UCA_MODE=usb`. Override via `--force-host` flag or re-running
    in `--mode host`. This is the strongest enforcement of "no host
    writes after setup."
- **Smoke-tested in both modes** тАФ `_uca_install_root` correctly returns
  `~/.config/usb-compute-automation` in host mode and
  `<persistence-mount>/usb-hemlock/etc/uca/` in USB mode (verified against
  a synthetic persistence dir tree + the real `/media/drdeek/Ventoy`).
