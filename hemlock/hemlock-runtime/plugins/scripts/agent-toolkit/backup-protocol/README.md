## How It Works

**Flow:**
1. Timer triggers every 6 hours
2. `backup.sh` runs
3. Pulls latest from GitHub (in case another machine pushed)
4. Copies critical files from `~/.openclaw/agents/` to `<RUNTIME_ROOT>/backups/agents/`
5. Encrypts sensitive files (.env, auth.json, .secrets/) with openssl AES-256-CBC
6. Syncs Docker config and plugins
7. Commits changes to local git repo
8. Pushes to GitHub via SSH deploy key

**What's NOT backed up (by design):**
- `checkpoints/` — 14GB+ of transient state, can be regenerated
- `sessions/` — transient conversation logs
- `logs/` — regenerable
- `skills/` — reinstallable from ClawHub
- `__pycache__/`, `node_modules/`, `.git/` — build artifacts

**Total backed up:** ~127MB (vs 17GB raw)

## Encryption

### How It Works

- **Algorithm:** AES-256-CBC with PBKDF2 key derivation
- **Key file:** `<RUNTIME_ROOT>/backups/.backup-key` (random 32-byte passphrase, base64 encoded)
- **Files encrypted:** `.env`, `auth.json`, anything in `.secrets/`
- **Files NOT encrypted:** config.yaml, SOUL.md, memory, projects (not sensitive)

### The Key File

```
<RUNTIME_ROOT>/backups/.backup-key
```

- Generated by `backup.sh --init`
- 32 random bytes, base64 encoded
- chmod 600 (owner read/write only)
- NEVER committed to git (in .gitignore)

**CRITICAL: Back this up somewhere safe.** Without it, encrypted files cannot be decrypted. Options:
- Password manager (1Password, Bitwarden, etc.)
- USB drive in a safe
- Encrypted note on your phone
- Print it out and store physically

### Check If Key Is Present
```bash
ls -la <RUNTIME_ROOT>/backups/.backup-key
```

### Manually Encrypt a File
```bash
openssl enc -aes-256-cbc -salt -pbkdf2 \
  -pass file:<RUNTIME_ROOT>/backups/.backup-key \
  -in plaintext-file \
  -out encrypted-file.enc
```

### Manually Decrypt a File
```bash
openssl enc -d -aes-256-cbc -pbkdf2 \
  -pass file:<RUNTIME_ROOT>/backups/.backup-key \
  -in encrypted-file.enc \
  -out decrypted-file
```

### Decrypt All .env Files
```bash
for dir in <RUNTIME_ROOT>/backups/agents/*/; do
  agent=$(basename "$dir")
  if [ -f "${dir}/.env.enc" ]; then
    openssl enc -d -aes-256-cbc -pbkdf2 \
      -pass file:<RUNTIME_ROOT>/backups/.backup-key \
      -in "${dir}/.env.enc" \
      -out "/tmp/${agent}-env-decrypted"
    echo "Decrypted: /tmp/${agent}-env-decrypted"
  fi
done
```

### Decrypt All auth.json Files
```bash
for dir in <RUNTIME_ROOT>/backups/agents/*/; do
  agent=$(basename "$dir")
  if [ -f "${dir}/auth.json.enc" ]; then
    openssl enc -d -aes-256-cbc -pbkdf2 \
      -pass file:<RUNTIME_ROOT>/backups/.backup-key \
      -in "${dir}/auth.json.enc" \
      -out "/tmp/${agent}-auth-decrypted.json"
    echo "Decrypted: /tmp/${agent}-auth-decrypted.json"
  fi
done
```

### Decrypt All .secrets Files
```bash
for dir in <RUNTIME_ROOT>/backups/agents/*/.secrets/; do
  for enc_file in "$dir"/*.enc; do
    [ -f "$enc_file" ] || continue
    name=$(basename "$enc_file" .enc)
    agent=$(echo "$dir" | sed 's|.*/agents/\([^/]*\)/.*|\1|')
    openssl enc -d -aes-256-cbc -pbkdf2 \
      -pass file:<RUNTIME_ROOT>/backups/.backup-key \
      -in "$enc_file" \
      -out "/tmp/${agent}-secret-${name}"
    echo "Decrypted: /tmp/${agent}-secret-${name}"
  done
done
```

### Change the Encryption Key

If you want to rotate the key or switch to a different passphrase:

```bash
# 1. Decrypt everything with the old key
cd ~/.openclaw-backup
for f in $(find . -name '*.enc'); do
  openssl enc -d -aes-256-cbc -pbkdf2 \
    -pass file:<RUNTIME_ROOT>/backups/.backup-key \
    -in "$f" -out "${f%.enc}.plaintext"
done

# 2. Generate new key
mv <RUNTIME_ROOT>/backups/.backup-key <RUNTIME_ROOT>/backups/.backup-key.old
openssl rand -base64 32 > <RUNTIME_ROOT>/backups/.backup-key
chmod 600 <RUNTIME_ROOT>/backups/.backup-key

# 3. Re-encrypt with new key
for f in $(find . -name '*.plaintext'); do
  openssl enc -aes-256-cbc -salt -pbkdf2 \
    -pass file:<RUNTIME_ROOT>/backups/.backup-key \
    -in "$f" -out "${f%.plaintext}.enc"
  rm -f "$f"
done

# 4. Commit
git add -A
git commit -m "key rotation: $(date -u +%Y-%m-%dT%H:%M:%SZ)"
git push

# 5. Store new key safely, delete old key
echo "New key stored. Old key at .backup-key.old — delete when sure."
```

### Change Encryption Algorithm

Edit `backup.sh` and change the `ENCRYPT_CMD` and `DECRYPT_CMD` variables:

```bash
# Current (AES-256-CBC):
ENCRYPT_CMD="openssl enc -aes-256-cbc -salt -pbkdf2 -pass file:${KEY_FILE}"
DECRYPT_CMD="openssl enc -d -aes-256-cbc -pbkdf2 -pass file:${KEY_FILE}"

# Alternative: ChaCha20 (faster on some hardware)
ENCRYPT_CMD="openssl enc -chacha20 -salt -pbkdf2 -pass file:${KEY_FILE}"
DECRYPT_CMD="openssl enc -d -chacha20 -pbkdf2 -pass file:${KEY_FILE}"

# Alternative: AES-256-CTR (stream cipher, no padding)
ENCRYPT_CMD="openssl enc -aes-256-ctr -pbkdf2 -pass file:${KEY_FILE}"
DECRYPT_CMD="openssl enc -d -aes-256-ctr -pbkdf2 -pass file:${KEY_FILE}"

# Alternative: Use age instead of openssl (install age first)
# ENCRYPT_CMD="age -r $(cat ${KEY_FILE})"
# DECRYPT_CMD="age -d -i ${KEY_FILE}"
```

After changing, you must re-encrypt all existing files (see "Change the Encryption Key" above).

### Use GPG Instead of OpenSSL

```bash
# Generate GPG key
gpg --full-generate-key

# In backup.sh, replace encrypt/decrypt:
ENCRYPT_CMD="gpg --batch --yes --passphrase-file ${KEY_FILE} -c"
DECRYPT_CMD="gpg --batch --yes --passphrase-file ${KEY_FILE} -d"
```

## Backup Schedule (Timer)

### Current Schedule
- **Every 6 hours** via systemd user timer
- **5 minutes after boot** (catches changes during downtime)
- **Persistent** (runs missed backups on wake)

### View Timer Status
```bash
systemctl --user list-timers openclaw-backup.timer
```

Output shows: next run time, last run time, how long until next.

### Change the Interval

Edit the timer file:
```bash
nano ~/.config/systemd/user/openclaw-backup.timer
```

**Every hour:**
```ini
[Timer]
OnBootSec=5min
OnUnitActiveSec=1h
Persistent=true
```

**Every 12 hours:**
```ini
[Timer]
OnBootSec=5min
OnUnitActiveSec=12h
Persistent=true
```

**Daily at 3 AM:**
```ini
[Timer]
OnBootSec=5min
OnCalendar=*-*-* 03:00:00
Persistent=true
```

**Twice daily (6 AM and 6 PM):**
```ini
[Timer]
OnBootSec=5min
OnCalendar=*-*-* 06,18:00:00
Persistent=true
```

**Every 30 minutes:**
```ini
[Timer]
OnBootSec=2min
OnUnitActiveSec=30min
Persistent=true
```

After changing:
```bash
systemctl --user daemon-reload
systemctl --user restart openclaw-backup.timer
```

### Trigger a Backup Manually
```bash
# Via script
<RUNTIME_ROOT>/backups/backup.sh

# Via systemd
systemctl --user start openclaw-backup.service
```

### Disable the Timer
```bash
systemctl --user stop openclaw-backup.timer
systemctl --user disable openclaw-backup.timer
```

### Re-enable the Timer
```bash
systemctl --user enable openclaw-backup.timer
systemctl --user start openclaw-backup.timer
```

### View Backup Logs
```bash
journalctl --user -u openclaw-backup.service --since "1 day ago"
```

```bash
for f in config.yaml SOUL.md AGENTS.md USER.md TOOLS.md MEMORY.md HEARTBEAT.md agent.json CUSTOM_FILE.md; do
```

### Add a New Directory to Backup

Edit `backup.sh`, find the directory sync section (around line 125):

```bash
# Current:
for d in memory projects tools archives knowledge submissions; do

# Add a new directory:
for d in memory projects tools archives knowledge submissions custom_dir; do
```

### Add a New Encrypted File

Edit `backup.sh`, find the encryption section (around line 135):

```bash
# Current:
for sensitive in .env auth.json; do

# Add:
for sensitive in .env auth.json custom-secret.json; do
```

### Exclude a File That's Currently Backed Up

Remove it from the `for f in ...` loop in the backup script.

### Exclude a Directory That's Currently Backed Up

Remove it from the `for d in ...` loop in the backup script.

### Add an Entirely New Source Directory

Add a new section in `backup.sh` after the Docker/plugins sections:

```bash
# ── Sync custom directory ────────────────────────────────────────
CUSTOM_DIR="${HOME}/path/to/custom"
if [ -d "$CUSTOM_DIR" ]; then
    mkdir -p "${BACKUP_DIR}/custom"
    rsync -a --delete \
        --exclude='node_modules' --exclude='__pycache__' \
        "${CUSTOM_DIR}/" "${BACKUP_DIR}/custom/" 2>/dev/null && changed=$((changed + 1))
fi
```

### Add Files to .gitignore

Edit `<RUNTIME_ROOT>/backups/.gitignore`:

```
# Add patterns to exclude:
*.large-file
temp/
cache/
```

### Remove Files from Backup on GitHub

```bash
cd ~/.openclaw-backup
git rm path/to/file
git commit -m "remove: path/to/file"
git push
```

The file will still exist in git history. To fully purge from history:
```bash
git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch path/to/file' \
  --prune-empty --tag-name-filter cat -- --all
git push --force
```

## Changing the Encryption Method

### Switch to Age (Modern, Simpler)

1. Install age:
```bash
sudo apt install age
```

2. Generate age key:
```bash
age-keygen -o <RUNTIME_ROOT>/backups/.age-key
chmod 600 <RUNTIME_ROOT>/backups/.age-key
```

3. Get the public key:
```bash
grep "public key:" <RUNTIME_ROOT>/backups/.age-key | cut -d: -f2 | tr -d ' '
```

4. Edit `backup.sh`, replace encrypt/decrypt commands:
```bash
AGE_PUB_KEY="<your-public-key>"
ENCRYPT_CMD="age -r ${AGE_PUB_KEY}"
DECRYPT_CMD="age -d -i ${HOME}/.openclaw-backup/.age-key"
```

5. Re-encrypt all files (see "Change the Encryption Key" above).

### Switch to GPG

1. Generate GPG key:
```bash
gpg --full-generate-key
```

2. Edit `backup.sh`:
```bash
GPG_RECIPIENT="your-email@example.com"
ENCRYPT_CMD="gpg --batch --yes -r ${GPG_RECIPIENT} -e"
DECRYPT_CMD="gpg --batch --yes -d"
```

### Disable Encryption Entirely

Edit `backup.sh` and replace the encryption section to just copy files:

```bash
# Instead of encrypting:
for sensitive in .env auth.json; do
    if [ -f "${agent_dir}/${sensitive}" ]; then
        cp "${agent_dir}/${sensitive}" "${dest}/${sensitive}"
        changed=$((changed + 1))
    fi
done
```

**WARNING:** This puts raw API keys and tokens in the GitHub repo. Only do this if the repo is private AND you trust GitHub's security.

## Restoring Data

### Restore a Single Agent
```bash
<RUNTIME_ROOT>/backups/backup.sh --restore
# Enter agent name when prompted
```

### Restore All Agents
```bash
<RUNTIME_ROOT>/backups/backup.sh --restore
# Enter "all" when prompted
```

### Restore to a Different Location
```bash
# Set AGENTS_DIR to a different path
AGENTS_DIR=/path/to/restore/location <RUNTIME_ROOT>/backups/backup.sh --restore
```

### Restore from a Specific Git Commit
```bash
cd ~/.openclaw-backup

# View history
git log --oneline

# Check out a specific commit
git checkout abc1234

# Run restore from that point
./backup.sh --restore

# Go back to latest
git checkout main
```

### Manual Restore (Without the Script)
```bash
# Copy config files
cp <RUNTIME_ROOT>/backups/agents/allman/config.yaml ~/.openclaw/agents/allman/
cp <RUNTIME_ROOT>/backups/agents/allman/SOUL.md ~/.openclaw/agents/allman/

# Decrypt and copy .env
openssl enc -d -aes-256-cbc -pbkdf2 \
  -pass file:<RUNTIME_ROOT>/backups/.backup-key \
  -in <RUNTIME_ROOT>/backups/agents/allman/.env.enc \
  -out ~/.openclaw/agents/allman/.env
chmod 600 ~/.openclaw/agents/allman/.env

# Copy memory directory
rsync -a <RUNTIME_ROOT>/backups/agents/allman/memory/ ~/.openclaw/agents/allman/memory/
```

### Restore After Total Data Loss
```bash
# 1. Clone the backup repo
git clone git@github.com:drdeeks/openclaw-backup.git ~/.openclaw-backup

# 2. Set up deploy key
# Copy your private key to ~/.ssh/id_ed25519_openclaw_backup
chmod 600 ~/.ssh/id_ed25519_openclaw_backup

# 3. Set up encryption key
# Copy your backup key to <RUNTIME_ROOT>/backups/.backup-key
chmod 600 <RUNTIME_ROOT>/backups/.backup-key

# 4. Restore all agents
cd ~/.openclaw-backup
./backup.sh --restore
# Enter "all"

# 5. Recreate Docker containers
cd ~/.openclaw/docker
docker compose up -d
```

```bash

  -f ~/.ssh/id_ed25519_openclaw_backup_new \

  -C "openclaw-backup-deploy-key-v2"
```

```bash
gh repo deploy-key add ~/.ssh/id_ed25519_openclaw_backup_new.pub \
```

```bash
# Edit backup.sh, find:
```

## Troubleshooting

### "Permission denied (publickey)"
```bash
# Test the deploy key
ssh -i ~/.ssh/id_ed25519_openclaw_backup -o IdentitiesOnly=yes git@github.com

# Should show: "Hi drdeeks/openclaw-backup! You've successfully authenticated"
```

### "No encryption key found"
```bash
# Check if key exists
ls -la <RUNTIME_ROOT>/backups/.backup-key

# If missing, generate a new one (will need to re-encrypt all files)
openssl rand -base64 32 > <RUNTIME_ROOT>/backups/.backup-key
chmod 600 <RUNTIME_ROOT>/backups/.backup-key
```

### "Push failed"
```bash
# Check remote is correct
cd ~/.openclaw-backup
git remote -v

# Pull first (in case of conflicts)
git pull --rebase

# Push again
git push
```

### Backup Takes Too Long
```bash
# Check what's being synced
du -sh ~/.openclaw/agents/*/memory/
du -sh ~/.openclaw/agents/*/projects/

# Exclude large directories by removing them from the sync loop in backup.sh
```

### Timer Not Running
```bash
# Check timer status
systemctl --user list-timers

# Check if timer is enabled
systemctl --user is-enabled openclaw-backup.timer

# Check logs
journalctl --user -u openclaw-backup.service --since "1 day ago"

# Restart
systemctl --user daemon-reload
systemctl --user restart openclaw-backup.timer
```

### "Decryption Failed" on Restore
```bash
# Verify key is correct
cat <RUNTIME_ROOT>/backups/.backup-key

# Try manual decryption
openssl enc -d -aes-256-cbc -pbkdf2 \
  -pass file:<RUNTIME_ROOT>/backups/.backup-key \
  -in <RUNTIME_ROOT>/backups/agents/allman/.env.enc \
  -out /tmp/test-decrypt

# If that fails, the key is wrong or the file is corrupted
# Check git history for a working version
cd ~/.openclaw-backup
git log --oneline -- agents/allman/.env.enc
git checkout abc1234 -- agents/allman/.env.enc
```

### Repo Too Large
```bash
cd ~/.openclaw-backup

# Check size
du -sh .git/

# Check what's large
git rev-list --objects --all | git cat-file --batch-check='%(objecttype) %(objectname) %(objectsize) %(rest)' | sed -n 's/^blob //p' | sort -rnk2 | head -10

# Prune
git reflog expire --expire=now --all
git gc --aggressive --prune=now
```

```bash
# Delete local backup
rm -rf ~/.openclaw-backup

# Delete remote repo
gh repo delete drdeeks/openclaw-backup --yes

# Start over
gh repo create drdeeks/openclaw-backup --private
git clone git@github.com:drdeeks/openclaw-backup.git ~/.openclaw-backup

<RUNTIME_ROOT>/backups/backup.sh --init
```
